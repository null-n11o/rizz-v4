## 画面構成

Expo Router のファイルベースルーティングを採用。ルートグループで認証画面とメイン画面を分離している。

```
app/
├── _layout.tsx          # ルートレイアウト（認証状態に応じてリダイレクト）
├── index.tsx            # 初期リダイレクト → /(auth)/login
├── +not-found.tsx       # 404ページ
├── (auth)/              # 認証グループ（未ログイン時のみアクセス可）
│   ├── _layout.tsx
│   ├── login.tsx        # ログイン画面
│   └── signup.tsx       # サインアップ画面
└── (tabs)/              # メインタブグループ（ログイン済み時のみアクセス可）
    ├── _layout.tsx      # タブナビゲーション定義
    ├── index.tsx        # ホーム画面（カウンター + プログレスバー）
    ├── statistics.tsx   # 統計データ画面 ❌ プレースホルダー
    ├── goal-settings.tsx # 目標設定画面
    └── settings.tsx     # 設定画面
```

---

## 各画面の詳細

### /(auth)/login — ログイン画面

- メールアドレス・パスワード入力
- ログインボタン
- サインアップ画面へのリンク
- Formik + Yup によるバリデーション

### /(auth)/signup — サインアップ画面

- ユーザーネーム・メールアドレス・パスワード入力
- サインアップボタン
- ログイン画面へのリンク
- Formik + Yup によるバリデーション

### /(tabs)/index — ホーム画面

- 今日の日付・ユーザー名グリーティング表示
- 4種類のカウンターボタン（コンパクトモード）
  - 声かけ数（approached）
  - 連絡先ゲット数（get_contact）
  - 即日デート数（instant_date）
  - 即数（instant_cv）
- 各カウンターのプログレスバー（現在値 / 日次目標）
- リロードボタン（目標・記録の再取得）
- ログアウトボタン
- オフライン・エラー状態の表示

### /(tabs)/statistics — 統計データ画面 ❌ 未実装

- **現状**: プレースホルダーのみ
- **予定機能**:
  - 日次/週次/月次/年次タブ切替
  - 各期間のグラフ表示（Victory Native）
  - 声かけ数・連絡先獲得率・即率などの指標表示
  - 期間ナビゲーション（前後の期間へ移動）

### /(tabs)/goal-settings — 目標設定画面

- 期間タイプ選択（日・週・月・年）
- 各指標の目標値入力
  - 声かけ目標数
  - 連絡先ゲット目標数
  - 即日デート目標数
  - 即目標数
- 保存ボタン
- デバッグメニュー（開発用）

### /(tabs)/settings — 設定画面

- **プロフィール設定**
  - ユーザーネーム変更
  - X(旧Twitter)URL 変更
  - プロフィール画像 ❌ 未実装
- **アカウント設定**
  - パスワード変更
- **アプリ設定**
  - テーマ切替（ライト/ダーク）
  - 言語切替（日本語/英語）

## ボトムタブバー

| タブ | ルート | アイコン |
|------|--------|---------|
| ホーム | `/(tabs)/` | home |
| 統計 | `/(tabs)/statistics` | bar-chart |
| 目標設定 | `/(tabs)/goal-settings` | target |
| 設定 | `/(tabs)/settings` | settings |

---

## 画面遷移フロー

```
アプリ起動
    ↓
認証状態チェック
    ├── 未ログイン → /(auth)/login
    │       ↓ 成功
    │   /(tabs)/index
    │
    └── ログイン済み → /(tabs)/index
```
